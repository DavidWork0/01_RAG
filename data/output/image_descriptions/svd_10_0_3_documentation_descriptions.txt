The image illustrates two different backup strategies: Incremental Cumulative Backup and Delta Backup.

### Incremental Cumulative Backup:
- **Full Backup (Sunday):** A full backup is performed on Sunday, creating a complete copy of the data.
- **Subsequent Days (Mon to Sat):** Each day, an incremental backup is made. These backups only include changes since the last backup, which could be the full backup or a previous incremental backup.
- **Sunday (Next Week):** Another full backup is performed, similar to the first Sunday. This ensures that even if data is lost, a complete backup is available.

### Delta Backup:
- **Full Backup (Sunday):** A full backup is performed on the first Sunday, creating a complete copy of the data.
- **Subsequent Days (Mon to Sat):** Each day, a delta (or differential) backup is made. These backups include all changes since the last full backup, not just incremental changes since the previous backup.
- **Sunday (Next Week):** Another full backup is performed, similar to the first Sunday, ensuring a complete copy of the data is available.

### Key Differences:
- **Incremental Cumulative Backup:** Each incremental backup includes all changes since the last backup, which could be full or incremental. This method requires more storage space over time but can restore data more efficiently by combining multiple incremental backups.
- **Delta Backup:** Each delta backup includes all changes since the last full backup. This method is simpler in terms of storage but requires more full backups over time to restore complete data.
